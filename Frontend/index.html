How They Work

Three Approaches
to Memory Freedom

01
🔢

Reference Counting

Objects carry a counter that increments/decrements as references are created or destroyed. Zero means dead.

Pros
✓ Immediate reclamation
✓ No GC pauses
✓ Predictable
Cons
✗ Cyclic references leak
✗ Counter overhead

Most Common
02
🔍

Mark & Sweep

Two-phase: trace all reachable objects from GC roots (mark), then sweep memory to free the unmarked ones.

Pros
✓ Handles cycles
✓ No per-write cost
✓ Complete collection
Cons
✗ Stop-the-world pauses
✗ Fragmentation
Deep dive →
03
🧬

Generational GC

Exploits the "generational hypothesis" — most objects die young. Separates heap into generations for efficient collection.

Pros
✓ Very fast in practice
✓ Short minor GC pauses
✓ High throughput
Cons
✗ Complex to implement
✗ Major GC still pauses
